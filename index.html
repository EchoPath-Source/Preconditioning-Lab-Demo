What this shows

It shows the product surface
  • route decisions become legible,
  • boundary and fallback state remain visible,
  • Harp class enriches repair semantics,
  • consumers use one source-agnostic bridge contract.
It does not claim
  • kernel #12 is solved,
  • production runtime performance,
  • private kernel exposure,
  • public SDK or physics validation.

Boundary

product consumer → kernel adapter / route-card → bridge event stream → overlay/debug UI. Product layers do not import Preconditioning Lab research internals, Don source material, or private Q-RRG kernel internals.
